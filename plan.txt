# MyFlix

### 영화 및 도서 추천 어플리케이션

---

### 개요

- 프로젝트명 : MyFlix
- 프로젝트 목적 : 국비지원 코딩교육 리액트과목 1차 프로젝트 평가
- 어플리케이션 설명 : 영화 및 도서 추천 어플리케이션
- 코딩 난이도 : 국비지원 코딩교육으로 한달정도 리액트 교육을 받은 정도의 실력을 가지고 있음
    - 배우지 않은 기술이나 프레임워크나 라이브러리는 사용하기 힘듬

---

### 개발환경

- React
    - react-router-dom와 axios 정도만 사용
    - Next.js나 TypeScript 같은 다른 라이브러리나 프레임워크는 사용하지 않음
- css
    - 마찬가지로 tailwind같은 프레임워크나 라이브러리 사용하지 말고 일반적인 css로만 구성.
    - 인라인 방식을 지양하고 가급적이면 .css파일을 만들어서 외부파일 방식으로 import하여 사용할것
- 백엔드(DB)
    - 오직 리액트만 사용
    - DBMS나 Java같은 백엔드 환경 사용하지않음
    - LocalStorage나 SessionStorage 사용함
- API
    - API Key 관리는 .env 파일을 별도로 생성해서 관리
    - 영화DB API : TMDB
        - 선정적인영화나 선정적인 포스터 필터링해야함
        - TMDB_API_KEY = “10a5e25315ab48e8dabb83995d5b7e11”
        - 한국 지역을 지원하는 영화 위주로 필터링
            - 한글로 번역이 안되어있는 영화라던지 이런건 필터링해야함
        - 영화 포스터가 없거나 설명(overview)같은게 없거나 예고편이나 티저영상같은게 없으면 필터링해야함
    - 도서DB API : 네이버
        - NAVER_CLIENT_ID = "x6PkPd_EEQgzdFRcxKxl"
        NAVER_CLIENT_SECRET = "gflwNj4Ze0"
        - 영화와 마찬가지로 한국 지역을 지원하는 도서 위주로 필터링
            - 한글로 번역이 안되어있는 도서는 필터링해야함
        - 영화와 마찬가지로 도서 소개글이나 도서 소개 이미지같은게 없다면 필터링해야함
- 디렉토리
    - 업무별 용도별 디렉토리도 구분을 잘지어서 관리할것
- 상수관리
    - 하드코딩이나 상수를 사용해야하는 로직은 적극적으로 사용하고 상수만 따로 관리하는 파일도 만들것

### 유저모드 세부기능

- **Header**
    - 구조(좌측부터)
        - MyFlix Logo 이동 버튼
        - 영화(Tap) 전환 버튼
        - 도서(Tap) 전환 버튼
        - 회원가입 이동 버튼
        - 로그인 이동 버튼
        - [등록한회원이름] 마우스호버 드롭박스
            - 마이페이지 이동 버튼
            - 위시리스트 이동 버튼
            - 추천 받은 컨텐츠 이동 버튼
            - 내가 만든 추천 리스트 이동 버튼
            - 내가 좋아요한 추천 리스트 이동 버튼
            - 회원수정 이동 버튼
    - 설명
        - 영화/도서 컨텐츠의 UI, 구조, 프로세스는 똑같고 Header에 영화 탭과 도서 탭으로 전환이 가능

- **Nav**
    - 구조(상단부터)
        - MyFlix 설명 문구
            - 당신만의 특별한 컨텐츠를 찾아보세요
            - 최신 영화부터 베스트셀러 도서까지, MyFlix에서 모든 것을 발견하세요
        - 검색바
            - 검색아이콘(검색바 텍스트 검색) 버튼
            - 상세아이콘(검색하지않고 검색화면으로 바로 이동) 버튼
        - MyFlix 선정 오늘의 추천 컨텐츠 이동 버튼, 다른 회원의 추천 리스트 보러가기 이동 버튼

- **Footer**
    - 구조(상단부터)
        - MyFlix
        ****영화와 도서를 사랑하는 사람들을 위한 추천 플랫폼
        최신 영화부터 베스트셀러 도서까지, 여러분의 취향에 맞는 컨텐츠를 추천해드립니다.

        Movie data from **TMDB** | Book data from **Naver Open API**

- **컨텐츠**
    
    ![image.png](attachment:26ca4030-5e06-4147-88fe-3e9a2795650d:image.png)
    
    - 구조(상단부터 세로 길이 형태)
        - 영화/도서 이미지
        - 설명(길어지면 ... 으로 생략)
        - 하트아이콘(위시리스트 등록/해제) 버튼
        - 추천아이콘(친구에게 추천하기) 모달 오픈
        - 리스트아이콘(내가 만든 추천 리스트) 모달 오픈
        - 그 외에 나머지 영역 클릭 시 컨텐츠 상세 조회 이동
    - 설명
        - 영화/도서를 간단하게 단독으로 보여주는 컨텐츠 단위
        - 홈페이지 내에 모든 갤러리 형태의 목록 조회 화면에 이 형식으로 나열
        - 하트 아이콘은 등록 버튼을 클릭하거나 이미 등록된 상태라면 아이콘 모양이 등록되어 있는 모양으로 변경 되어야하고 반대로 등록된 버튼을 다시 클릭하면 원래 아이콘으로 롤백
        - 공유아이콘 클릭 시 내가 등록한 친구 회원 목록 모달이 오픈되고 다중 선택으로 추천할수있음
        - 리스트아이콘 클릭 시 내가 등록한 내가 만든 추천 리스트 목록 모달이 오픈되고 목록 중 선택
        - 해당 형식을 사용하여 조회하는 화면 리스트
            - 위시리스트
            - 추천 받은 컨텐츠
            - 검색 화면

- **컨텐츠 상세**
    - 구조(상단부터)
        - 다른 영화 추천 버튼(새로고침)
        - 포스터/이미지
        - 제목
        - 설명
        - 세부내용
        - 영화의 경우 예고편/티저 영상
        - 나의 별점과 리뷰
    - 설명
        - 위에 설명한 컨텐츠의 아이콘 버튼이 아닌 영역 클릭 시에 이 화면으로 이동
        - 내가 등록한 별점과 리뷰가 있다면 같이 출력
        - 홈 화면에 MyFlix 선정 오늘의 추천 컨텐츠 버튼 클릭 시에도 이 화면으로 오는데 인기 TOP 200 중 랜덤으로 조회한다.
            - TOP 200 랜덤 추천 컨텐츠는 한글로 번역이 되어있는 컨텐츠여야하고 제목과 설명이 있어야하며 영화의 경우 예고편/티저 영상이 존재해야한다.

- **Home**
    - 메인배너(최신 인기 컨텐츠 10개 캐러셀 전환)
    - 다른 유저들의 추천 리스트 10개 목록 조회

    - 회원가입
    - 로그인

- **Header > 마이페이지**
    - 마이페이지
        - 구조(상단부터)
            - 위시리스트(간략) 더보기 이동 버튼
            - 추천 받은 컨텐츠(간략) 더보기 이동 버튼
            - 내가 만든 추천 리스트(간략) 더보기 이동 버튼
            - 내가 좋아요한 추천 리스트(간략) 더보기 이동 버튼
        - 설명
            - 내가 등록한 컨텐츠들과 추천 리스트들을 간략하게 한눈에 볼 수 있는 화면
    - 위시리스트
        - 구조
            - 내가 등록한 위시리스트 컨텐츠 목록 조회
            - 내가 리뷰한 컨텐츠만 보기 체크 박스
            - 10개씩 조회되고 페이징
    - 추천 받은 컨텐츠
        - 구조
            - 추천 받은 컨텐츠 목록 조회
            - 확인 하지 않은 컨텐츠만 보기 체크 박스
            - 10개씩 조회되고 페이징
        - 설명
            - 추천 받은 컨텐츠 중 내가 확인 하지 않은 컨텐츠라면 빨간점(뱃지) 표시
            - 한번이라도 컨텐츠 상세 화면으로 갔다면 확인한것으로 간주
    - 내가 만든 추천 리스트
        - 구조
            - 내가 만든 추천 리스트 목록 조회
            - 좋아요 받은 숫자 조회
            - 내가 만든 추천 리스트 등록/수정/삭제
            - 내가 만든 추천 리스트 목록 중 하나를 클릭 시에 내가 만든 추천 리스트 조회 화면으로 이동
        - 설명
            - 다른 유저들에게 추천 해줄 내가 만든 추천 컨텐츠 리스트들을 관리하는 화면
            - 멜론의 플레이리스트 처럼 나만의 추천 리스트를 “혼자 보기 아까운 나만의 영화 10선!”, “쓸쓸한 가을에 보기 좋은 영화”, “스트레스 확 풀리는 시원한 액션 영화 추천!”, “쌀쌀한 겨울 따뜻한 가족 소설 10선!” 등 컨셉 혹은 나만 아는 명작 컨텐츠들을 공유하는 시스템
            - 내가 만든 추천 리스트를 하나라도 등록 해놨다면 모든 컨텐츠는 내가 만든 추천 리스트 컨텐츠에 등록 할 수있다.
            - 다른 유저들이 나의 추천 컨텐츠를 좋아요했다면 좋아요 받은 숫자가 올라간다.
    - 내가 좋아요한 추천 리스트
        - 구조
            - 다른 유저의 추천 리스트를 좋아요 클릭 한 추천 리스트 목록 조회
            - 좋아요 받은 숫자 조회
            - 다른 유저의 추천 리스트 목록 중 하나를 클릭 시에 다른 유저의 추천 리스트 조회 화면으로 이동
        - 설명
            - 내가 좋아요를 준 다른 유저의 추천 컨텐츠 리스트들을 조회하는 화면
            - 좋아요 버튼을 다시 클릭 시에 내가 좋아요한 추천 리스트에 제외 된다.
